## Обзор проекта

Проект представляет собой модификацию клиента **MU Online Season 3 Final**, реализованную в виде динамической библиотеки (`Main.dll`). Библиотека инжектируется в основной процесс игры и расширяет его функциональность через перехват функций (hooking) с использованием **Microsoft Detours**.

**Основные возможности:**
- Расширенные игровые механики (система Rage, ресеты, ранги, ежедневные награды)
- Улучшенная графика (камера, туман, свечение, частицы)
- Кастомный пользовательский интерфейс (меню, мини-карта, настройки)
- Многоуровневая система античита
- Расширенный сетевой протокол с XOR-шифрованием
- Система автоподключения и управления сессией

## Архитектура

Клиент построен по модульной архитектуре с единой точкой входа в виде DLL:

```
┌──────────────────────────────────────────────────────────┐
│               MU Online Client (Main.dll)                │
│              Инжектируемый DLL-модуль                     │
└────────────────────┬─────────────────────────────────────┘
                     │
         ┌───────────┼───────────┐
         │           │           │
    ┌────▼───┐  ┌────▼───┐  ┌───▼────┐
    │Сетевой │  │Игровая │  │Графич. │
    │ слой   │  │ логика │  │движок  │
    └───┬────┘  └───┬────┘  └───┬────┘
        │           │           │
   ┌────┴────┐ ┌────┴────┐ ┌───┴─────┐
   │Protocol │ │Rage Sys │ │OpenGL   │
   │XOR Enc  │ │Reset    │ │Camera   │
   │Packet   │ │Ranking  │ │Texture  │
   │TCP Sock │ │Items    │ │Effects  │
   └────┬────┘ └────┬────┘ └───┬─────┘
        │           │           │
   ┌────┴───────────┴───────────┴──────┐
   │     Менеджер состояния игры       │
   │   (хуки памяти, объекты, карты)   │
   └────────────┬──────────────────────┘
                │
    ┌───────────┼───────────┐
    │           │           │
┌───▼──┐  ┌────▼───┐  ┌────▼───┐
│Анти- │  │Ввод и  │  │Отладка │
│чит   │  │управл. │  │и логи  │
└──────┘  └────────┘  └────────┘
```

**Ключевые принципы:**
- **Hook-based Architecture** — перехват функций игрового движка через Microsoft Detours
- **Direct Memory Access** — чтение/запись по захардкоженным адресам памяти для доступа к состоянию игры
- **Модульность** — каждая подсистема реализована в отдельных `.h/.cpp` файлах
- **XOR-шифрование** — пакеты шифруются 32 XOR-ключами

## Основные модули

### 1. Ядро и точка входа

#### `dllmain.cpp` — Точка входа DLL
- Обрабатывает событие `DLL_PROCESS_ATTACH` при загрузке библиотеки
- Инициализирует все подсистемы клиента
- Устанавливает хуки на функции игрового движка

#### `stdafx.h` — Предкомпилированные заголовки
- Подключает системные библиотеки: `<windows.h>`, `<gl/gl.h>`, `<gl/glu.h>`, `<winsock2.h>`
- Подключает STL-контейнеры: `<string>`, `<vector>`, `<list>`, `<unordered_map>`
- Подключает процессные утилиты: `<Psapi.h>`, `<tlhelp32.h>`

#### `Define.h` — Глобальные константы
- Адрес сервера, версия клиента, название окна
- Флаги включения/выключения функций (Rage, античит, крылья, телепортация и т. д.)
- Макросы цветов для UI

#### `Structure.h` — Структуры данных
- Определения основных игровых структур
- Структуры объектов, персонажей, предметов

---

### 2. Игровые механики

#### `RageSystem.h/cpp` — Система навыков Rage
- **36 навыков** с прогрессией по уровням
- Структура `RageUser` хранит уровень, очки, опыт и 46+ параметров:
  - Бонус урона, критический шанс, отражение урона
  - Скорость атаки, бонус оружия, превосходный урон
  - Бонус крыльев, призыв питомца
- Каждый навык (`RageSkill`) имеет имя, начальный урон, максимум очков, процентный/абсолютный режим

#### `ResetSystem.h/cpp` — Система ресетов
- Сброс персонажа с сохранением прогресса
- Награды за ресет: деньги (`NeedMoney`), очки (`RewardPoint`), кредиты, WCoin
- Возможность выдачи предметов в качестве награды
- Настраиваемые требования: уровень (`NeedLevel`), максимальное число ресетов (`MaxReset`)

#### `GrandReset.h` — Гранд-ресет
- Расширенная версия ресета с повышенными наградами

#### `AdvancedStats.h/cpp` — Расширенная статистика
- Дополнительные параметры персонажа, отсутствующие в базовом клиенте
- Расширенная система подсчёта характеристик

#### `DailyReward.h/cpp` — Ежедневные награды
- Система ежедневных бонусов за вход в игру
- Настраиваемые награды по дням

#### `Ranking.h/cpp` — Рейтинговая система
- Таблица лидеров **Top 100**
- Структура `CharTop` хранит: имя, класс, уровень, ресеты, гранд-ресеты, характеристики, гильдию, премиум-статус
- Сортировка по различным критериям

#### `DungeonSiege.h/cpp` — Осада подземелья
- Гильдовое PvE-событие
- Подсчёт очков, управление состоянием события
- Структуры: `PMSG_DUNGEON_SIEGE_SCORE`, `PMSG_DUNGEON_SIEGE_STATE`

#### `Items.h` — Управление предметами
- Базовая система предметов и инвентаря

#### `ItemStack.h/cpp` — Стакирование предметов
- Объединение одинаковых предметов в стопки
- Управление лимитами стакирования

#### `ItemValue.h/cpp` — Стоимость предметов
- Динамический расчёт цен покупки и продажи
- Цветовая маркировка предметов по редкости:
  - `White (0x00)` — обычные
  - `Blue (0x01)` — магические
  - `Red (0x02)` — редкие
  - `Gold (0x03)` — легендарные
  - `Green (0x04)` — уникальные
  - `Violet (0x06)` — эпические
  - `DarkViolet (0x0B)` — мифические

#### `ItemRank.h/cpp` — Ранги предметов
- Система рангов и уровней качества предметов

#### `Shop.h/cpp` — Магазин
- До 120 предметов в ассортименте (`MAX_ITEM_SHOP_PRICE`)
- Расчёт цен покупки (`GetBuyPrice`) и продажи (`GetSellPrice`)
- Структура `SHOP_INFO`: индекс, уровень, прочность, опции, цена

#### `Jewels_Wings.h/cpp` — Ювелирные камни и крылья
- Обширная система кастомизации экипировки
- Настройка атрибутов крыльев и камней

#### `JewelsBank.h/cpp` — Банк камней
- Хранилище ювелирных камней
- Управление депозитом и снятием

#### `LuckyWheel.h/cpp` — Колесо удачи
- Система случайных наград

### 4. Графика и рендеринг

#### Используемый API: OpenGL 1.x

Клиент использует прямые вызовы OpenGL для рендеринга: `glEnable`, `glClearColor`, `glTexParameter` и др.

#### `Visual.h/cpp` — Визуальные эффекты
- Управление визуальными эффектами персонажей и окружения

#### `CameraEx.h/cpp` — Расширенная камера
- Структура `CAMERA_ZOOM` — параметры зума (мин/макс проценты и лимиты)
- Структура `CAMERA_ADDR` — адреса памяти для управления камерой (Zoom, RotY, RotZ, PosZ)
- Класс `CCamera` — управление зумом, вращением, динамическим позиционированием

#### `Fog.h/cpp` — Система тумана
- Динамический туман с цветовой картой
- Настройка плотности и дальности

#### `Glow.h/cpp` — Эффект свечения
- Подсветка предметов и персонажей
- Цветовое свечение в зависимости от типа объекта

#### `Texture.h/cpp` — Загрузка текстур
- Загрузка изображений форматов: **JPG**, **TGA**, **BMP**
- Функции: `LoadImagesJPG()`, `LoadImagesTGA()`, `LoadFilesBMD()`

#### `BMDLoad.h/cpp` — Загрузка 3D-моделей
- Загрузка моделей в проприетарном формате **BMD** (MU Online)

#### `HPBar.h/cpp` — Полоски здоровья
- Отрисовка полосок HP/SD (здоровье/щит) для игроков и монстров
- Управление viewport-ом для корректного отображения

#### `WeaponFloat.h/cpp` — Парящее оружие
- Визуальный эффект левитирующего оружия персонажа

### 5. Сетевой протокол

#### `Protocol.h/cpp` — Определения протокола
- Заголовки пакетов:
  - `PBMSG_HEAD` — однобайтовый размер, один код
  - `PBMSG_HEAD2` — однобайтовый размер, код + подкод
  - `PWMSG_HEAD2` — двухбайтовый размер, код + подкод (для больших пакетов)
- Основные типы сообщений:
  - `PMSG_UPDATECHARACTER` — обновление характеристик
  - `PMSG_LEVELUP` — повышение уровня
  - `PMSG_CHARINFOMAIN` — информация о персонаже
  - `PMSG_UPDATEPOINTS` — обновление очков
  - `PMSG_SHOP_PRICE_LIST` — прайс-лист магазина
  - `PMSG_TPLIST` — список телепортов
  - `PMSG_MONSTERHP_VIEWPORT` — HP монстров
  - `PMSG_PROPENSITY_VIEWPORT` — наклонности

#### `Packet.h/cpp` — Движок пакетов
- Класс `CStreamPacketEngine_Client`:
  - `Clear()` — очистка буфера
  - `AddData()` — добавление данных в пакет
  - `ExtractPacket()` — извлечение пакета
- **XOR-шифрование** с 32 ключами (`MAX_XOR_KEYS = 32`)

#### `Packets.h/cpp` — Структуры пакетов
- Определения всех типов пакетов для обмена с сервером

#### `ExTeleport.h/cpp` — Расширенная телепортация
- Кастомные точки телепортации
- Управление списками локаций

#### `Reconnect.h/cpp` — Переподключение
- Автоматическое восстановление соединения при разрыве

#### `User.h/cpp` — Данные пользователя
- Управление данными персонажа и учётной записи
- Структуры: `PMSG_TARGETDATA_REQ/ANS`, `PMSG_BATTLE_LIST`, `CHAR_UPDATEINFO`

### 6. Безопасность и античит

#### `AntiCheat.h/cpp` — Основная система античита (203 КБ)
Самый крупный модуль проекта. Реализует комплексную защиту:
- **Мониторинг процессов** — обнаружение запущенных чит-утилит
- **Проверка целостности памяти** — контроль модификаций кода
- **Обнаружение файлов** — проверка наличия подозрительных файлов (CMain.dll и др.)
- **Защита окна** (`TitleWindowMain`) — контроль заголовка окна
- **Y-Protection** (`YProtectionMain`) — дополнительный уровень защиты
- **Чёрный список** — база известных чит-программ

#### `AntiCheatPlus.h/cpp` — Расширенный античит
Дополнительные функции защиты:
- `PMSG_AH_SUB`, `PMSG_AH_CONFIG` — конфигурация античита
- `CG_PMSG_HDDKEY` — верификация по ключу жёсткого диска (привязка к оборудованию)
- `PMSG_CRCBase` — проверка CRC-сумм
- `PMSG_UPDATEFILE` — механизм обновления файлов
- `ANITHACK_PROCDUMP` — обнаружение дампов процесса

#### `Protect.h/cpp` — Защита кода
- Защита целостности кода в памяти
- Обнаружение модификаций

#### `Heurictic_Detect.cpp` — Эвристическое обнаружение
- Поведенческий анализ для обнаружения новых читов

#### `CCRC32.h/cpp`, `Crc321.h/cpp`, `Crc32Load.h/cpp` — Система CRC32

## Сборка проекта

### Требования
- **IDE:** Visual Studio 2010 или новее
- **Платформа:** Win32 (32-bit)
- **Зависимости:** входят в состав проекта (Detours) или являются системными (OpenGL, Winsock2)

### Процесс сборки

1. Откройте `Client.sln` в Visual Studio
2. Выберите конфигурацию:
   - **Debug** — без оптимизаций, с отладочной информацией
   - **Release** — максимальная оптимизация (`/O2`), function-level linking, COMDAT folding
3. Соберите решение: **Build → Build Solution** (или `Ctrl+Shift+B`)
4. Результат: файл `Main.dll` в корневом каталоге проекта

### Связываемые библиотеки

| Библиотека | Назначение |
|------------|------------|
| `ws2_32.lib` | Windows Sockets 2 (сетевое взаимодействие) |
| `Opengl32.lib` | OpenGL (графический рендеринг) |
| `glu32.lib` | OpenGL Utilities |
| `psapi.lib` | Process Status API (мониторинг процессов) |
| `detours2.lib` | Microsoft Detours v2 (перехват функций) |
| `detoured.lib` | Библиотека Detoured |

## Развёртывание

1. Скомпилируйте проект → получите `Main.dll`
2. Поместите `Main.dll` в каталог игрового клиента MU Online
3. DLL загружается при запуске игры и автоматически расширяет функциональность клиента
4. Убедитесь, что сервер доступен по адресу, указанному в `Define.h`
